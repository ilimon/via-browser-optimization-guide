Introduction

This guide walks you through building a high-performance, privacy-focused ad-blocking setup on Via Browser for Android. The configuration is optimized for mid-range devices like the Redmi 12 running Android 15, but applies to most Android phones.

What You'll Achieve
  • Block ads and trackers without slowing your device
  • Achieve a perfect 100/100 score on adblock detection tests
  • Noticeably faster browsing and reduced data usage
  • Maintain compatibility with mainstream websites
Prerequisites
  • Via Browser (latest version)
  • Access to WebView DevTools (Settings > Advanced > WebView DevTools)
  • 10–15 minutes of setup time

Part 1 — Filter Configuration

1.1 Access Ad Blocker Settings

  1. Open Via Browser
  2. Tap the menu icon (≡) → Settings
  3. Navigate to Ad Blocker → Filter Settings
  4. You'll see your current filter list subscription area

1.2 Remove Redundant Filters

Disable built-in filters as they are trash. If previously installed, Start by clearing existing filters to eliminate overlap and reduce device strain.

Why remove redundant filters?

Multi-filter redundancy wastes memory and CPU on your Redmi 12. A lean, curated list performs better than a bloated one.

Remove these (if present):

  • https://raw.githubusercontent.com/StevenBlack/hosts/master/hosts Reason: Blunt domain-blocking approach; massive overlap with HaGeZi Pro+
  • Multiple regional filters (unless you browse those regions regularly)
  • Duplicate EasyList variants

1.3 Add the Core Filter Stack

Add these filters in order:

Filter URL Purpose
EasyList https://easylist.to/easylist/easylist.txt Core ad blocking
uBlock filters https://cdn.jsdelivr.net/gh/uBlockOrigin/uAssets@master/filters/filters.txt Enhanced ad detection
HaGeZi's Pro+ DNS https://raw.githubusercontent.com/hagezi/dns-blocklists/main/adblock/pro-plus.txt Privacy + tracker blocking
EasyPrivacy https://easylist.to/easylist/easyprivacy.txt Behavioral tracker blocking
Fanboy's Annoyance https://secure.fanboy.co.nz/fanboy-annoyance.txt Social widgets + cookie notices

To add each filter:

  1. In Filter Settings, tap + Add Filter
  2. Paste the URL
  3. Tap Save
  4. Repeat for all five filters

1.4 Verify Filter Load

  1. Go to Ad Blocker → Filter Statistics
  2. You should see ~100,000+ rules loaded across all filters
  3. If the number is significantly lower, re-check your URLs for typos

Expected result: Fast page loads with zero perceptible slowdown on a Redmi 12.

Part 2 — WebView DevTools Optimization

WebView is the rendering engine behind Via Browser. Optimizing its flags improves performance dramatically.

2.1 Access WebView Flags

  1. Open Via Browser
  2. Tap menu (≡) → Settings → Advanced
  3. Tap WebView DevTools
  4. You'll see the WebView Flags interface

2.2 Enable Core Performance Flags

Search for and enable each flag below:

Flag Name Status Purpose
ignore-gpu-blocklist Enable Forces GPU acceleration (safe on Redmi 12)
WebViewSurfaceControl Enable Modern rendering pipeline
WebViewVulkanIntermediateBuffer Enable Vulkan graphics stability
WebViewEnableADPF Enable Adaptive performance framework
UseDMSAForTiles Enable Faster tile rendering during scroll
VSyncDecoding Enable Syncs video decoding with display refresh

How to enable a flag:

  1. Tap the search box at the top
  2. Type the flag name
  3. Tap the flag result
  4. Toggle Enabled
  5. Return to the list

2.3 Add Advanced Optimization Flags

These flags provide marginal but noticeable improvements:

Flag Name Status Purpose
enable-features=WebViewThreadSafeMedia Enable Improved media handling stability
enable-features=WebViewNewGestureHandling Enable Better touch responsiveness

To add a custom flag:

  1. Tap + Add Flag (if available) or use the search bar
  2. Enter the flag name exactly as shown
  3. Toggle Enabled

2.4 Disable Privacy-Draining Flags

Disable or leave off these flags to protect privacy:

Flag Name Status Reason
AI service provider settings Disable Reduces unnecessary telemetry to AI providers
Telemetry flags (if present) Disable Privacy protection

Part 3 — Via Browser Privacy Settings

3.1 Disable Built-in AI Features

  1. Open Via Browser
  2. Tap menu (≡) → Settings → General
  3. Find AI service settings
  4. Disable:
    • "AI service provider"
    • "AI prompt"
Why disable AI features?

These features send data to external AI providers. Disabling them improves privacy and reduces background data usage.

3.2 Enable Core Privacy Options

  1. In Settings, go to Privacy & Security
  2. Enable:
    • "Do Not Track" requests
    • "Clear data on exit" (cookies, cache, history)
    • "Block third-party cookies"
    • "HTTPS-Only Mode" (if available)

3.3 Configure User Agent (Optional)

Recommendation: Keep the default user agent. This avoids breaking website compatibility while maintaining adequate privacy.

Only change if:

  • A specific website requires a different user agent
  • You're targeting a specific device profile (rare on mobile)

Part 4 — Testing & Verification

4.1 Run an Adblock Detection Test

  1. Visit https://adblock-tester.com/
  2. Run the full test suite
  3. Expected result: 100/100 or near-perfect score
If you score below 95/100:
  • Verify all filters loaded correctly (Part 1, Step 1.4)
  • Check that WebView flags are enabled (Part 2)
  • Clear browser cache: Settings → Privacy → Clear Browsing Data → All Time

4.2 Test Real-World Browsing

Visit these sites to confirm functionality:

  • YouTube – Should load video ads correctly; should block sidebar ads
  • Reddit – Should load normally; should remove promotional content
  • News sites (BBC, CNN, etc.) – Should load articles with minimal ads
  • Bangladeshi sites (if applicable) – Should load without geo-blocking issues

If a site breaks: The ad blocker may be too aggressive. See Part 5 for whitelist instructions.

Part 5 — Troubleshooting

Issue: Website Won't Load or Shows Errors

Solution:

  1. Tap menu (≡) → Ad Blocker
  2. Tap the toggle to temporarily disable the ad blocker
  3. Reload the page
  4. If it loads, the blocker is the culprit

Permanent fix:

  1. Re-enable the ad blocker
  2. Go to Ad Blocker → Whitelist
  3. Add the domain (e.g., example.com)
  4. Reload the page

Issue: Slow Browsing Despite Filter Optimization

Checklist:

  • Close unused tabs (reduces memory load)
  • Disable WebView experimental flags one by one to isolate the culprit
  • Restart Via Browser
  • Clear cache: Settings → Privacy → Clear Browsing Data

Issue: Ads Still Appearing

Solution:

  1. Verify filter load count in Ad Blocker → Filter Statistics (should be 100k+)
  2. Run the adblock-tester.com test again
  3. Check filter URLs for typos
  4. Some ads (video pre-rolls on YouTube) require additional userscript solutions

Part 6 — Maintenance Schedule

Weekly

  • Check Filter Statistics to ensure rules are loading
  • Test on one major website (YouTube, Reddit, etc.)

Monthly

  • Run adblock-tester.com to verify score
  • Review WebView flags; disable any causing slowdowns
  • Clear browser cache and temporary files

Every 3 Months

  • Review filter URLs; update any that have moved
  • Check Via Browser for updates; apply new versions
  • Reassess which filters you actually need (remove unused ones)
